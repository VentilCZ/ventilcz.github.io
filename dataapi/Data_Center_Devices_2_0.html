Attribute label Attribute name Attribute type Purpose
Agent Software Version agent_software_version String (40) Defines software version of agent application
Attached Devices attached_devices String (40) Defines attached device(s) to UPS battery
Audible Alarm Status audible_alarm_status Integer Defines the audible alarm status
Available Slots u_available_slots Integer Count of free slots in particular HW enclosure
Billable u_billable True/False Flag for Smart Billing customer invoicing (entify whether CI is billable to customer or not)
CI number u_number String (40) Public unique identifier of each record in CMDB
Classification u_subclass Reference (CI Classification) CMDB logical model CI type which is mapped to particular CI Class used in actual CMDB physical model
Company company Reference (Company) The Company to whom the CI is assigned to. Companies describe an organizational hierarchy of customer/vendor to a level of detail seen necessary for IT service management purposes.
Correlation ID correlation_id String (512) External ID of the CI used in the external primary data source provider
Cost center cost_center Reference (Cost Center) Reference to Tieto internal Cost center
Created sys_created_on Date/Time Timestamp of CI creation
Created by sys_created_by String (40) User that created record
Customer Project u_customer_project Reference (TERP Project) Customer project identification for Smart Billing cost forwarding
Customer Task u_customer_task Reference (Terp Task) Customer task identification for Smart Billing cost forwarding
Customer invoicing group u_customer_invoicing_group String (18) Customer cost centre information or other grouping factor for the invoice
Customer invoicing reference 1 u_customer_invoicing_ref_1 String (255) Customer internal invoicing/financial information: e.g. customer Cost Center
Customer invoicing reference 2 u_customer_invoicing_ref_2 String (255) Customer internal invoicing/financial information: e.g. customer Country/Org Unit
Decomission Date u_decomission_date Date/Time Defines date when CI was decomissioned
Description short_description String (1,000) Optional free text to describe important aspects of the CI, information that is not possible to store in attributes / KB and other related CIs via relationships. This attribute is visible to customers.
Discovery source discovery_source String (40) Primary data source provider name
Enclosure u_enclosure Reference (Hardware Enclosure) Reference to the HW enclosure to which slot belongs to
Environment u_environment Choice Describes if the CI is related to production or some other use
Exact location u_exact_location String (100) Free text information about physical location within a building
Firmware version u_firmware_version String (40) Hardware version
GL account gl_account String (80) Customer internal invoicing/financial information
Installed install_date Date/Time Timestamp when CI first time taken in use
Installed HW u_installed_hw Reference (Server Hardware) References the installed HW (Blade Server) which this slot is occupied by
Last discovered last_discovered Date/Time Timestamp of the last update from primary data source provider
Lifecycle install_status Integer Lifecycle shows current CI lifecycle phase. See the linked knowledge article for more information.
Location location Reference (Location) Physical location of CI (as in most detail as possible)
MAC Address mac_address String (100) Contains the list of all MAC addresses attached to the device (LAN, WAN, WLAN)
Maintenance schedule maintenance_schedule Reference (Schedule) Reference to CI specific maintenance windows slot
Manufacturer manufacturer Reference (Company) Reference to Company which represents the manufacturer of the CI
Model ID model_id Reference (Product Model) Reference to Model entity (part of Asset Management)
Model number model_number String (255) Contains the vendor sales item product name. The syntax is defined by the device vendor in their product catalog.
Mount side u_mount_side Choice Describes whether the device is mounted on the front or back of the rack
Name name String (255) The name of the CI. It should uniquely identify the CI for a human. This is not forced as a unique field on database level, but set as unique on the user interface for defined CI classes (Server, Network Gear etc).
Nominal Input Voltage (RMS VAC) nom_input_volt Floating Point Number Defines input voltage (can be different in Europe and America)
Nominal Output Voltage (RMS VAC) nom_output_volt Floating Point Number Defines output voltage
Primary Business Service u_primary_bs Reference (Automated Business Service) Reference to primary Business Service which is used for Event Management (for automated created incidents)
Provider SO u_provider_so Reference (Service Offering) Reference to provider Service Offering related to this CI
Rack u_rack Reference (Rack) Reference to Rack where is this rackable hardware (any HW CIs that can be racked: Server Hardware, Netgear, Storage Devices etc.) mounted on. If this reference exists, then it consumes Rack space.
Serial number serial_number String (255) Serial number of device
Slot status u_slot_status Choice Defines status of the slot
Sys ID sys_id Sys ID (GUID) System unique identifier of each record in CMDB
U count u_count Decimal Describes the mount height from the "U start" position for the device
U start u_start Decimal The starting position of physical device in the rack. Most bottom position is 1.
Updated sys_updated_on Date/Time System timestamp of CI modification (changed content of CI -> reflects CI staleness)
Updated by sys_updated_by String (40) User that modified record
